# UI Theme Prep

这个目录是三套新 UI 的预研包，暂时不接入当前项目运行入口。当前应用的默认蓝白 UI 保持不变，后续测试完成后可以把这里的主题变量、素材位和切换逻辑整体迁入。

## 主题方向

1. `wild-kingdom`：荒野遗迹冒险
   - 对应参考：开放世界奇幻冒险、古代机关、荒野营地、装备收集。
   - 关键感受：探索、发现、试炼、收集。

2. `ember-fellowship`：远征史诗奇幻
   - 对应参考：电影感高奇幻远征、古地图、符文、旅队、暗影敌人。
   - 关键感受：任务、同行、对抗、成长。

3. `pocket-future`：口袋未来伙伴
   - 对应参考：圆润儿童动画、未来小道具、明亮课堂、陪伴式机器人伙伴。
   - 关键感受：轻松、奖励、道具、陪伴。

## 版权处理原则

不直接使用或复刻受版权保护的原作人物、怪物、武器、道具、标志、独特图案和名称。这里准备的是“题材和体验层面的原创二创”：让孩子感受到相近的冒险/史诗/未来伙伴氛围，但素材本身是新的。

## 文件说明

- `theme-spec.md`：三套主题的 UI 思路、组件映射和接入计划。
- `recognition-cue-matrix.md`：把参考作品的强识别元素转译成原创元素的边界表。
- `component-skin-map.md`：当前项目所有主要 UI 组件的主题化设计地图。
- `asset-prompt-pack.md`：后续生成背景图、角色图、怪物图、道具图时可直接使用的提示词。
- `theme-tokens.css`：未来可接入项目的主题变量草案。
- `theme-preview.html`：独立静态预览页，可直接用浏览器打开，不依赖项目代码。
- `deep-theme-board.html`：更完整的组件级主题设计板。
- `asset-manifest.json`：主题素材位、文件路径和后续生成状态。
- `assets/*.svg`：每套主题的原创图标符号和背景草图。

## 后续接入建议

1. 在现有设置页增加“界面风格”选择项，默认值为 `default`。
2. 在 `state.settings` 中保存 `uiTheme`，写入已有 profile storage。
3. 在页面初始化和主题切换时设置：`document.body.dataset.uiTheme = state.settings.uiTheme`。
4. 引入主题 CSS 覆盖层，优先只覆盖颜色、背景、按钮、卡片、奖励图标，不动练习逻辑。
5. 最后再替换奖励图、任务卡图标、背景插画等更重的视觉素材。
