# Component Skin Map

这份文件把当前项目的主要 UI 都映射成主题化组件。核心原则：主题不是换色，而是“页面世界观 + 材质 + 形状 + 图标 + 文案 + 反馈”的统一系统。

## 全局层级

视觉优先级：

1. 题目和答案最高：背景和纹理必须低对比。
2. 主操作按钮第二：只有一个高强调按钮。
3. 当前任务状态第三：任务卡和进度条要明显但不能抢题目。
4. 装饰元素最低：只放边缘、角标、空状态和奖励位。

统一控制：

- `body`：主题世界背景。
- `.app-shell`：内容舞台，不做厚重外框。
- `.topbar`：主题身份最强的区域。
- `.question-panel`：主题材质最完整的主面板。
- `.choice`：主题细节最丰富，但必须保证可读。

## 单词选项边框

选项是孩子每天点击最多的组件，需要比普通按钮更主题化。

### 荒野遗迹

- 选项 1：叶纹盾角，左上角有叶片扣。
- 选项 2：石板角，右上角有小裂纹。
- 选项 3：青铜装备格，底边有铆钉。
- 选项 4：符文槽，右下角有发光菱形。
- Hover：边框出现短促青绿光线。
- Correct：四角符文同时亮起。
- Wrong：角落出现红色裂纹，不大面积染红。

### 远征史诗

- 选项 1：蜡封角，左上角圆封。
- 选项 2：铁角护片，右上角金属扣。
- 选项 3：路线针角，底部有路线小点。
- 选项 4：火种角，右下角小火焰。
- Hover：像地图被灯照到，中心稍亮。
- Correct：火种沿边框走一圈。
- Wrong：暗影只从一个角侵入，保留文字可读。

### 口袋未来

- 选项 1：星星贴纸角。
- 选项 2：胶囊按钮角。
- 选项 3：抽屉标签角。
- 选项 4：小扳手贴纸角。
- Hover：按钮轻微上浮，边缘变亮。
- Correct：星章弹出，绿色只做点缀。
- Wrong：红色维修提示条，不能像错误惩罚。

## 按钮系统

每套主题至少三类按钮，不能只是同一个圆角按钮换颜色。

### 荒野遗迹

- Primary：发光符文石按钮，八角切边。
- Secondary：木牌按钮，浅棕底，绳结边。
- Danger：裂纹红石按钮，只在清空记录等危险操作使用。

### 远征史诗

- Primary：锻造金属 + 火光按钮。
- Secondary：羊皮纸卷轴按钮。
- Danger：暗红封蜡按钮。

### 口袋未来

- Primary：胶囊按钮，左侧蓝白，右侧红色动作端。
- Secondary：贴纸按钮，白底蓝边。
- Danger：软红维修按钮，圆润但明显。

## 页面级设计

### 练习页

练习页是主题最完整的页面：

- 顶栏体现世界观。
- 筛选区变成“选择任务范围”的工具。
- 任务卡变成四种不同任务入口。
- 题目区变成主题核心装置。
- 选项区四个边框不重复。
- 侧栏变成背包/日志/抽屉。

### 错词本

错词本不应只是列表：

- 荒野：错词像“未清理营地”，每行有小怪标记。
- 远征：错词像“地图上的暗影标记”，可用旗标/路线点。
- 口袋未来：错词像“待维修小票”，每行有修理状态。

### 统计页

统计页要主题化但保持家长可读：

- 荒野：统计卡是石碑，周图是营火/精力柱。
- 远征：统计卡是里程碑，周图是路线旗。
- 口袋未来：统计卡是仪表，周图是电量柱。

### 设置页

设置页低频但也要统一：

- 荒野：冒险手册条目。
- 远征：远征条例卷轴。
- 口袋未来：控制台开关和胶囊输入。

### 账户弹窗

弹窗应是主题容器，不只是默认白框：

- 荒野：石门打开的面板。
- 远征：摊开的卷轴。
- 口袋未来：弹出的道具盒。

## 接入优先级

第一阶段：CSS 皮肤

- 背景、顶栏、按钮、选项、题目面板、任务卡、侧栏。
- 不换数据结构，不换 JS 逻辑。

第二阶段：主题素材

- 背景图、品牌图标、奖励图、空状态图、错词怪物图。

第三阶段：文案主题化

- 根据主题切换任务名、奖励名和反馈语。
- 保留家长设置页的清晰描述，不要过度游戏化。

第四阶段：轻动画

- 正确：发光 / 火种 / 星章弹出。
- 错误：裂纹 / 暗影 / 维修提示。
- 每个动画 120-240ms，避免影响答题节奏。
