# Three Theme UI Spec

## 总体结构

当前项目适合做“默认主题 + 主题覆盖层”：

- 默认 UI：继续使用现有蓝白风格。
- 新主题：通过 `body[data-ui-theme="..."]` 覆盖 CSS 变量和少量组件样式。
- 主题选择：放在主界面右上角或设置页第一项。孩子使用时右上角更直接，家长管理时设置页更稳。
- 资产结构：每个主题都准备 `background`、`brand`、`mission`、`reward`、`choice`、`empty-state` 六类素材位。

建议主题枚举：

```js
const UI_THEMES = [
  { id: "default", label: "默认" },
  { id: "wild-kingdom", label: "荒野遗迹" },
  { id: "ember-fellowship", label: "远征史诗" },
  { id: "pocket-future", label: "口袋未来" }
];
```

## 1. 荒野遗迹冒险

### 视觉关键词

草地、风、古代石纹、青铜装备、发光符文、营地旗帜、天空岛碎片、试炼门。

### UI 映射

- 页面背景：淡绿色草坡、远山、浮石轮廓，低对比，不能干扰文字。
- 顶栏：木质/石质混合感，品牌图标用“发光符文石”。
- 标签页：像装备栏切换，选中态是青绿色发光边。
- 筛选工具栏：像冒险手册的筛选条，输入框为浅羊皮纸。
- 任务卡：四张任务卡分别映射为“今日试炼 / 新词采集 / 弱点营地 / 自由探索”。
- 题目面板：中央像试炼石板，英文单词是主要符文内容。
- 选项按钮：像小道具格，正确时绿光，错误时赤红裂纹。
- 奖励框：以原创装备、符文、徽章为奖励，不使用原作武器或徽章。

### 素材位

- 角色：披风少年侦察者、短发荒野学徒、护目镜机关师。
- 怪物：营地小怪、石壳守卫、夜光史莱姆。
- 武器/工具：风化短剑、叶纹盾、符文石板、滑翔布。
- 道具：精力果、发光矿石、古代齿轮、试炼钥石。

### 避免

不要使用 Link、Zelda、Ganon、Master Sword、Triforce、Sheikah eye、Korok 等可识别名称或图案。

## 2. 远征史诗奇幻

### 视觉关键词

古地图、火光、石墙、符文边框、旅队徽记、暗色山脊、银色月光、皮革书页。

### UI 映射

- 页面背景：远山地图和火光渐层，整体偏电影感但保持亮度。
- 顶栏：像远征日志标题栏，品牌图标用“旅队徽章”。
- 标签页：像地图上的章节签。
- 筛选工具栏：羊皮纸标签，选择框像卷轴输入。
- 任务卡：四张任务卡分别映射为“今日远征 / 新词招募 / 暗影清理 / 自由巡游”。
- 题目面板：像摊开的地图或石桌，题目区可加低对比符文边。
- 选项按钮：像旅队卡牌，正确时金绿光，错误时暗红裂纹。
- 奖励框：以原创徽章、地图碎片、符文火种为奖励。

### 素材位

- 角色：斗篷学习者、银袍导师、矮人工匠、森林弓手。
- 怪物：暗影面具兵、洞穴巨怪、黑雾骑士、碎石哨兵。
- 武器/工具：叶形短刃、旅人法杖、锻造锤、古地图。
- 道具：火种徽章、银叶扣、符文书页、远征罗盘。

### 避免

不要使用 Frodo、Gandalf、Aragorn、Legolas、Sauron、One Ring、Eye of Sauron、White Tree 等可识别名称或图案。

## 3. 口袋未来伙伴

### 视觉关键词

圆润、蓝白红黄、未来课堂、口袋道具、贴纸、星星、软质按钮、友好机器人伙伴。

### UI 映射

- 页面背景：明亮房间、桌面、贴纸、柔和网格，避免过度儿童化。
- 顶栏：像学习伙伴控制台，品牌图标用“口袋星章”。
- 标签页：像道具抽屉，选中态是红色小标签。
- 筛选工具栏：像文具盒/抽屉分区。
- 任务卡：四张任务卡分别映射为“今日充能 / 新词胶囊 / 错词修理 / 自由实验”。
- 题目面板：像未来学习板，题目更大、更圆润。
- 选项按钮：像道具按钮，正确时绿色星光，错误时红色软提示。
- 奖励框：以原创未来道具、星章、发音铃、记忆胶囊为奖励。

### 素材位

- 角色：圆形口袋机器人伙伴、铅笔帽小助手、笑脸学习终端。
- 怪物：作业云、瞌睡泡泡、错词小影子、乱序方块。
- 武器/工具：记忆灯、发音铃、词汇胶囊、星门卡片。
- 道具：口袋徽章、时间贴纸、纠错扳手、鼓励星。

### 避免

不要使用 Doraemon、Nobita、Take-copter、Anywhere Door、Time Machine、四次元口袋等可识别名称或设计。

## 主题切换接入点

建议最小代码变更：

1. `index.html`
   - 在顶栏账号选择旁增加一个 `select#themeSelect`。
   - 或在设置页增加一个 `setting-line`。

2. `app.js`
   - `defaultProfile().settings` 增加 `uiTheme: "default"`。
   - `dom` 增加 `themeSelect`。
   - 初始化时调用 `applyTheme(state.settings.uiTheme)`。
   - 监听 `themeSelect.change`，保存并应用。

3. `styles.css`
   - 保留现有 `:root` 默认变量。
   - 新增 `theme-tokens.css` 覆盖层。

建议函数：

```js
function applyTheme(theme) {
  document.body.dataset.uiTheme = theme && theme !== "default" ? theme : "";
  const color = getComputedStyle(document.body).getPropertyValue("--theme-color").trim();
  document.querySelector('meta[name="theme-color"]')?.setAttribute("content", color || "#1e88e5");
}
```

## 奖励系统改造思路

现有奖励类型 `star / goal / group` 可以扩展为主题化显示：

- `star`：默认星星；荒野为“试炼星石”；远征为“火种徽章”；口袋未来为“鼓励星章”。
- `goal`：默认奖杯；荒野为“精力果”；远征为“远征旗”；口袋未来为“充能电池”。
- `group`：默认进度环；荒野为“符文齿轮”；远征为“地图碎片”；口袋未来为“词汇胶囊”。

这样不改变奖励逻辑，只替换视觉表达。
